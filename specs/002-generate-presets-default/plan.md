# Implementation Plan: Generate Default Presets

**Branch**: `002-generate-presets-default` | **Date**: 2026-05-27 | **Spec**: `specs/002-generate-presets-default/spec.md`

**Input**: Feature specification from `specs/002-generate-presets-default/spec.md`; planning note: "We should have a few recommended models to choose from and pick them based on simple heuristic of the user's system"

---

## Summary

Add a new snap app, `se-llama.generate-presets`, that creates `$SNAP_USER_COMMON/config/presets.ini` on demand. The app selects a default preset from a curated set of recommended model profiles using simple local heuristics, writes a privacy-preserving human-editable config, and refuses to overwrite existing user config unless explicitly requested.

The implementation will follow the existing snap-local structure: a small shell app entrypoint under `snap/local/bin/` invokes a Python helper under `snap/local/libexec/`. The helper owns template generation, non-destructive file writes, resource detection, and CLI output.

---

## Technical Context

**Language/Version**: POSIX shell wrapper; Python 3 runtime helper; YAML snap metadata.

**Primary Dependencies**: Existing snapcraft build; Python 3 standard library only; existing llama.cpp `presets.ini` format consumed by `llama-server`.

**Storage**: `$SNAP_USER_COMMON/config/presets.ini` for generated config; `$SNAP_USER_COMMON/models/` referenced by generated model profiles; optional backup next to the config only when explicit replacement is requested.

**Testing**: Shell integration tests for snap app behavior; Python unit tests for heuristic/profile selection and file safety; snap lint remains required for packaging validation.

**Target Platform**: Ubuntu 24.04 LTS, amd64, strict snap confinement.

**Project Type**: Snap package with CLI apps.

**Performance Goals**: Preset generation completes in under 10 seconds; heuristic detection performs only local reads and no network calls.

**Constraints**: Must not download models; must not overwrite user config by default; must not enable persistent prompt or KV cache settings; must avoid broad filesystem permissions; generated config must remain valid for llama-server native preset parsing.

**Scale/Scope**: Single-user local inference setup; a small curated recommendation set of 3 starter profiles for low, balanced, and large local systems.

## Phase 0: Research Summary

Research completed in `research.md`. All unknowns resolved.

Key decisions:
- New snap app name: `generate-presets`, exposed as `se-llama.generate-presets`.
- CLI shape: default auto-generation, `--profile low|balanced|large|auto`, `--force`, and `--dry-run`.
- Heuristic: memory-first classification using locally readable memory information, with best-effort GPU detection only when available inside confinement.
- Recommendation set: 3 curated profiles: low, balanced, and large. They reference model repositories/files but do not download anything.
- File safety: default behavior refuses to overwrite. `--force` writes a timestamped backup before replacement.
